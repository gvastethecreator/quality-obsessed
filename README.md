Use it when the task calls for obsessive, evidence-backed quality instead of a competent first pass.

## Install

Install with the Skills CLI:

```powershell
npx skills add gvastethecreator/quality-obsessed
```

Or copy [`SKILLS/quality-obsessed`](./SKILLS/quality-obsessed) into your Codex skills directory.

## Core Contract

- Inspect the real artifact: runtime, screenshots, output, tests, states, or edge cases.
- Start skeptical: run adversarial result analysis before approval on broad, quality-sensitive, recovery, audit, or critique work.
- Change the artifact, not just the plan.
- Convert the strongest in-scope criticism into a real fix when the artifact is editable.
- Prove each quality claim with evidence.
- Research current, external, or repo facts automatically when they affect the quality bar, implementation, risk, or final claim.
- Map fog automatically when a mission is too uncertain for confident implementation, spec, or tickets.
- Improve adjacent product surface when it materially serves the same user outcome and stays within safe boundaries.
- Test through agreed public seams with independent expected values.
- Beat the baseline, or say `failed` / `red/failed`.
- If meaningful weaknesses remain and the user gave no limit: 30-loop floor, no hard cap, Loop 30 verdict (`continue`, `ask`, `stop`).
- Use council, grilling, delegated review, and visual horizons when they improve the artifact, decision, proof, or cuts.
