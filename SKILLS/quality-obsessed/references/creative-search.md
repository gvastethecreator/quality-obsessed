# Creative Search

Use this branch only when the user explicitly asks for creative, standout, greenfield, or direction-sensitive work. It is a direction-selection protocol, not a cosmetic polish pass. Routine fixes, bug repairs, and design-system/conformance work skip it.

## Machine-checkable contract

```text
creative_search: diverge-prototype-compare-commit
creative_trigger: explicit-creative-standout-greenfield-or-direction-risk
direction_count: three-materially-distinct
direction_distance: thesis-structure-or-behavior-not-cosmetic
prototype_policy: cheapest-representative-artifact
selection_basis: user-value-signature-feasibility-proof
hybrid_policy: no-default-hybrid
signature_move: one-memorable-useful-move
subtraction_move: remove-one-generic-or-diluting-element
blind_audience_read: brief-hidden-perception-test
blind_read_fields: understood-action-memory-mismatch
```

## Search loop

1. State the user outcome, constraints, and direction risk. If the request is only correctness, routine maintenance, or conformance, stop this branch and use the smallest normal path.
2. Generate exactly three feasible directions. Make the difference structural: each must change the thesis, information structure, or user behavior. Different colors, type, spacing, or component skins do not count.
3. Build the cheapest artifact that makes each direction judgeable: a sketch, one-screen flow, short storyboard, runnable slice, or representative fixture. Keep the same input and viewport where comparison matters.
4. Compare all three against user value, signature, feasibility, and proof. Record the chosen direction and why the others were rejected. Do not blend them by default; a hybrid needs an explicit evidence-backed reason.
5. Add one signature move that is memorable and useful, then subtract one generic or diluting element. The signature must improve the user's task, not add decoration.
6. Run a blind audience read without showing the brief or intended answer. Ask only: what is understood, what action seems next, what is remembered, and what feels mismatched. Resolve every material mismatch before committing; if the read cannot run, mark the gate limited rather than green.

## Direction record

```text
Direction A/B/C:
Thesis, structure, or behavior changed:
Representative artifact:
User value:
Signature opportunity:
Feasibility and proof path:
Rejected or selected:
```

The selected direction re-enters the normal Evidence and Pressure gates. Keep the three artifacts and blind-read notes as comparison evidence; do not certify the direction from a written rationale alone.
