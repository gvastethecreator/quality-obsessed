# Docs And Data Profile

Use this profile for documentation, specs, reports, analysis, datasets, dashboards, and read-only audits.

## Documentation and specs

Treat the document as the artifact. Inspect the reader's task, decision, next action, examples, constraints, acceptance criteria, caveats, and nearby contradictions. A documentation-only change can be a material improvement when it makes the contract more accurate, actionable, or verifiable.

Cross-check commands, flags, APIs, paths, and current claims against primary or repository-owned sources. Render or convert the document when layout is part of the claim.

## Data and reports

Start from the decision or question the artifact must support. Verify metric definitions, source lineage, data quality, missingness, filters, time windows, denominators, and caveats before polishing presentation.

Separate observed facts, calculations, interpretation, and recommendation. Expected values must come from a worked example, agreed definition, query result, fixture, or independent source.

## Audit mode

Audit the requested surface without changing the target. Findings include location, evidence, user or operator harm, likely cause, severity, recommended move, and proof needed. Open findings affect the artifact verdict; they do not turn a completed audit into a blocked task.

## Proof

Use exact source pointers, rendered output, validation commands, query results, reconciled totals, or before/after reader tasks. If current or external facts shape the conclusion, cite the primary source and identify uncertainty.
