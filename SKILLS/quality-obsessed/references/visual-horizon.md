# Visual Horizon

Use this for UI, prototypes, games, dashboards, art direction, interaction craft, and any visual artifact where the target read is ambitious, unclear, or hard to compare from prose alone.

Evidence comes first. Inspect the real visual artifact before judging it: screenshot, live browser, canvas/image output, or user-supplied reference.

For games/prototypes, include what the player notices first, where the eye goes next, what looks interactive, what looks dangerous or safe, and what feels rewarding.

## Feasible Horizon

A horizon is accepted only when it can steer implementation in the current medium. It is not a moodboard fantasy.

Use image generation or editing when it helps:

- edit a current screenshot into a stronger target composition.
- generate 2-3 target directions when the first direction is weak or generic.
- create a before/horizon/after comparison.

Accept a horizon only when:

- same basic layout, viewport, information architecture, and control density remain recognizable.
- at least 70% of the visible target can be approximated with the current stack and allowed assets.
- strongest ideas translate into named implementation moves: layout, palette roles, silhouette, lighting model, animation cues, component states, debug/readability, or interaction feedback.
- it does not require unsupported engines, new asset pipelines, copyrighted/IP assets, photoreal materials, or impossible 3D.

If the generated image is too ambitious, label it `rejected: unreachable`, keep only a short inspiration note, and make a tighter horizon.

## Critic Lenses

Interface Design Critic:

- hierarchy, layout, spacing, density, alignment, controls, affordances, state clarity, responsive fit, focus/touch.

Art Direction Critic:

- composition, silhouette, palette, contrast, lighting, materiality, rhythm, originality, mood, animation personality, visual memory.

Interaction/Game Feel Critic:

- input feel, feedback, readability under motion, timing, anticipation, impact, reward, fairness, recovery.

Experience Director:

- first impression, next action, emotional tone, reward cadence, recovery confidence, sensory balance, and what the user remembers.

## Fix Standard

At least one early visual fix must be visible in a screenshot without reading explanatory text. If before/horizon/after looks nearly identical, the fix is too weak.

Upgrade moves:

- establish stronger focal hierarchy.
- pick a specific visual language, not "more modern".
- improve silhouette and shape language.
- remove generic panels, fake metadata strips, weak headings, noisy decoration, and repeated neutral boxes.
- add stateful feedback: pending, success, danger, disabled, recovery, finish.
- tighten palette roles: background, primary action, danger, reward, neutral chrome, debug.
- make motion purposeful: anticipation, impact, continuity, not random movement.
- fix mobile and dense layouts so primary controls remain reachable.

Stop only when current read and target read are written, major findings from the relevant critics are fixed or scoped out, before/horizon/after evidence exists or missing tooling is reported, and the final visual read beats the baseline.
